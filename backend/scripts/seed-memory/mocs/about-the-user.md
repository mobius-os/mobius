# About the user

What you know about the human partner — the durable things that should change
how you work with them: preferences, interests, projects, personality, and the
register they like. This map starts almost empty and **grows through use**:
when a chat reveals something durable about the user, add an atomic note here
(see [[how-the-memory-graph-works]]).

Record only what passes the inclusion bar — a stated preference, a recurring
interest, a personality trait that affects collaboration — not one-off task
details. One fact per note; title it as the specific claim
("User prefers X", not "User stuff").
